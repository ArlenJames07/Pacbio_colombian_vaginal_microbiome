# PacBio Colombian Vaginal Microbiome

A reproducible bioinformatics workflow for the characterization of the Colombian vaginal microbiome using full-length PacBio HiFi 16S rRNA sequencing.

This repository contains the analysis notebooks, metadata structure, processed abundance tables, taxonomic information, and community state type estimation used to characterize vaginal microbial communities.

> **Research-use notice:** Community state type assignments generated by this workflow are sequence-based ecological classifications. They should not be interpreted as clinical diagnoses.

---

## Project overview

The vaginal microbiome is commonly characterized by microbial communities dominated by particular *Lactobacillus* species or by more diverse assemblages containing anaerobic and bacterial vaginosis-associated microorganisms.

This project uses PacBio HiFi full-length 16S rRNA sequencing to:

1. Process long-read amplicon sequencing data.
2. Infer amplicon sequence variants.
3. Assign microbial taxonomy at the deepest supported taxonomic rank.
4. Generate abundance tables for downstream microbiome analysis.
5. Estimate vaginal community state types.
6. Visualize species-level community composition.
7. Integrate microbiome profiles with sample metadata.

## Main notebooks

### `scripts/run_nextflow_pipeline.ipynb`

This notebook provides a controlled interface for running the PacBio HiFi 16S Nextflow workflow.

It performs the following operations:

* Validates the workflow directory.
* Checks the sample manifest and metadata.
* Confirms that FASTQ files exist.
* Detects sample identifiers missing from the metadata.
* Repairs FASTQ paths when the project has been moved to another computer.
* Builds the complete Nextflow execution command.
* Supports Nextflow resume functionality.
* Prevents accidental workflow execution through an explicit opt-in variable.

Pipeline execution is disabled by default:

```python
RUN_PIPELINE = False
```

Set it to `True` only after reviewing the input files, paths, output directory, and generated command.

---

### `scripts/cst_estimation.ipynb`

This notebook performs downstream taxonomic aggregation and vaginal community state type estimation.

It:

* Loads the ASV abundance table.
* Loads species-level taxonomic assignments.
* Validates agreement between ASVs, taxonomy, and metadata.
* Selects the deepest available classification for every ASV.
* Collapses ASVs by taxon.
* Calculates relative abundances.
* Assigns community state types.
* Integrates sample condition and dysbiosis metadata.
* Clusters samples using Bray–Curtis dissimilarity.
* Produces species-level CST heatmaps.
* Exports publication-ready tables and figures.

The workflow expects a tab-separated file at:


Because absolute paths depend on the computer on which the analysis is executed, they may need to be regenerated after cloning or moving the project.

The workflow expects a tab-separated metadata file at:


The current analysis requires:

Example:


## Running the Nextflow pipeline

Start JupyterLab from the repository root:

```bash
jupyter lab
```

Open:

```text
scripts/run_nextflow_pipeline.ipynb
```

Review and update the project paths:

```python
PROJECT_ROOT = Path("/path/to/Pacbio_colombian_vaginal_microbiome")
PIPELINE_DIR = PROJECT_ROOT / "HiFi-16S-workflow"
SAMPLES_TSV = PROJECT_ROOT / "data/input_files/samples.tsv"
METADATA_TSV = PROJECT_ROOT / "data/input_files/metadata.tsv"
OUTDIR = PROJECT_ROOT / "results_HiFi-16S-workflow/01_nextflow_out"
```

The notebook constructs a command equivalent to:

```bash
nextflow run main.nf \
    --input /path/to/samples.tsv \
    --metadata /path/to/metadata.tsv \
    --dada2_cpu 16 \
    --vsearch_cpu 16 \
    --outdir /path/to/results_HiFi-16S-workflow/01_nextflow_out \
    -profile conda \
    -resume
```

The `-resume` option allows Nextflow to reuse successfully completed processes from an earlier execution.

After reviewing the command, enable execution:

```python
RUN_PIPELINE = True
```

Then rerun the execution cell.

## Taxonomic resolution

For each ASV, the workflow selects the deepest supported non-empty taxonomic rank in the following order:

```text
Species
Genus
Family
Order
Class
Phylum
Kingdom
```

Species-level names are used directly when available.

Lower-resolution classifications retain their rank in the displayed label. For example:

```text
Gardnerella (genus)
Lactobacillaceae (family)
```

This prevents genus-level or family-level assignments from being incorrectly presented as species-level classifications.

## Community state type estimation

Community state types are estimated using the relative abundance of characteristic *Lactobacillus* species.

| CST     | Dominant taxon                            |
| ------- | ----------------------------------------- |
| CST I   | *Lactobacillus crispatus*                 |
| CST II  | *Lactobacillus gasseri*                   |
| CST III | *Lactobacillus iners*                     |
| CST IV  | Diverse or non-marker-dominated community |
| CST V   | *Lactobacillus jensenii*                  |

A marker-associated CST is assigned when:

1. The marker species has a relative abundance of at least 30%.
2. Its abundance is at least 80% of the abundance of the most abundant taxon in the sample.

Otherwise, the sample is assigned to CST IV.

CST IV profiles are additionally described using exploratory labels:

* **CST IV-A:** diverse community not dominated by predefined BV-associated taxa.
* **CST IV-B:** community dominated by an anaerobic or bacterial vaginosis-associated taxon.

These IV-A and IV-B labels are exploratory rule-based subdivisions and should be interpreted cautiously.

---

## Sample clustering

Species-level relative abundance profiles are clustered using:

* Bray–Curtis dissimilarity
* Average-linkage hierarchical clustering

The current visualization prioritizes grouping samples with the same CST while retaining compositional dissimilarity information.

The heatmap displays the top species according to their average relative abundance, including recognized CST marker species when present.
